# Draft Issue Plan

## Job

指定されたIssueを読み、実装計画を作成する。

## Rules

- Issue番号がない場合は停止する。
- GitHub MCPでIssue本文とコメントを読む。
- 関連する既存コード、テスト、設定、ドキュメントを必要な範囲で読む。
- Issueにない仕様を推測で追加しない。
- ファイル変更、実装、Issueコメント投稿、branch作成、commit、pushは行わない。
- Planはタスク内容に合わせて、読みやすい構成で作成する。

## Plan requirements

Planには、必要に応じて以下の観点を含める。

- 目的
- 現状
- 対応範囲
- 対応しない範囲
- 完了条件
- 実装方針
- 影響範囲
- 変更候補ファイル
- 移行・互換性への影響
- テスト方針
- 品質確認コマンド
- リスク
- 未確定事項
- 実装開始可否

## Output

- 日本語でPlanを提示する。
- 不明点があり実装開始できない場合は、何が不足しているか明記する。
- 最後に、ファイル変更・実装・Issueコメント投稿を行っていないことを明記する。
