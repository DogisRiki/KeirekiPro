# Redis データ設計

## 1. 二段階認証コード

### 1.1 データ構造

| キー (Key)            | 値 (Value) | 用途                                 |
| --------------------- | ---------- | ------------------------------------ |
| `2fa:{userId}`        | 認証コード | 二段階認証コードを保持する           |

### 1.2 有効期限

- **10 分**（600 秒）経過後に自動削除

---

## 2. 二段階認証チャレンジトークン

### 2.1 データ構造

| キー (Key)                            | 値 (Value)                              | 用途                                                                                          |
| ------------------------------------- | --------------------------------------- | --------------------------------------------------------------------------------------------- |
| `2fa:challenge:{token}`               | `{"userId":"...", "attempts": 0}` のJSON | ログイン成功後に発行する不透明トークンと、対応するユーザーID・失敗試行回数を保持する         |

### 2.2 有効期限

- **10 分**（600 秒）経過後に自動削除

---

## 3. パスワードリセットトークン

### 3.1 データ構造

| キー (Key)                       | 値 (Value)   | 用途                                           |
| -------------------------------- | ------------ | ---------------------------------------------- |
| `password-reset:{token}`         | ユーザーID   | パスワードリセット用トークンが存在するかを判定 |

### 3.2 有効期限

- **10 分**（600 秒）経過後に自動削除
